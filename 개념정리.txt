Express Framework
- 프레임워크 : 개발자에게 공통적으로 개발의 틀을 제공하는 환경
- 장점 : 개발의 속도가 향상, 유지보수가 편하다
- 단점 : 초기에 학습하기 어렵다.
- Express : 기존에 서버가 하는 역할이 너무 많아서 서버가 하는 역할을 분배하는 기법

[기본구조] -> 규칙
1) config : 설정에 관련된 파일만 관리하는 폴더 (DB연걸, API key값 관리)

2) public : 정적파일들을 관리하는 공간(css, js,img,video ..)

3) routes : 사용자의 요청에 맞게 역할을 분배하는 공간(이정표)

4) views : 동적페이지를 담당하는 공간 -> 페이지의 틀은 개발자 생성 -> 값은 컴퓨터가 설정

5) app.js : 실질적인 서버관련 코드를 작성하는 공간(컨트롤타워) -> 실질적인 서버

[Express 만드는 프로세스]
- 사용할 워크스페이스 생성
- Express 설치
- 4개의 기본폴더를 생성( 직접 생성 )
- app.js 제작
